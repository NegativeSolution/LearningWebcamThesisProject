# AI Webcam Object Recognition (CLIP-Based)

## Overview

This project is a local AI powered webcam application that can recognize objects using a modern open vocabulary vision model. Instead of being restricted to a fixed set of categories like many traditional classifiers, the system compares what the webcam sees against natural language descriptions of objects.

The system uses the CLIP approach where both images and text are converted into numerical embeddings. These embeddings are compared using similarity scoring to determine which label best matches the visual input.

The entire system runs locally on the user's computer. No cloud services, APIs, or internet connections are required for inference.

The program allows a user to:

1. Open their webcam
2. Draw a box around an object on screen
3. Have the AI guess what the object is
4. Save the cropped image along with the predicted label

This repository includes everything needed to:

* Build the project in Visual Studio
* Run the object recognition program
* Generate new AI label embeddings
* Store and organize captured images

---

# How the Program Works (Simple Explanation)

Humans learn by connecting words with visual experiences.

Example:

You see an object and someone tells you:

"That is a coffee mug."

Over time your brain connects:

visual pattern -> word meaning

This project works similarly.

The AI converts both images and text into numerical representations called embeddings.

image -> numerical embedding
text description -> numerical embedding

The program compares these embeddings using similarity scoring.

Whichever text description is most similar to the image embedding becomes the predicted label.

Example process:

Image from webcam
Convert image to embedding
Compare with embeddings for:

"a photo of a hammer"
"a photo of a laptop"
"a photo of a coffee mug"

The label with the highest similarity score wins.

When the project is opened, Visual Studio automatically restores required packages from NuGet.

Users do not need to manually install packages.

---

# Data Folder

```
Data/
   labels.txt
   text_embeddings.json
```

This folder defines the vocabulary the AI can recognize.

### labels.txt

A list of text descriptions such as:

a photo of a laptop
a photo of a coffee mug
a photo of a screwdriver

Each line represents a possible object category.

---

### text_embeddings.json

This file stores the numerical embeddings for each label.

The program compares the webcam image embedding against these vectors to determine the most similar label.

This file is automatically generated by the Python script.

---

# Models Folder

```
Models/
   clip_image_encoder.onnx
```

This folder contains the CLIP image encoder model.

The ONNX file is a portable neural network format used by ONNX Runtime.

It converts images into numerical embeddings that can be compared with text embeddings.

---

# Scripts Folder

```
Scripts/
   generate_embeddings.py
```

This script converts text labels into embeddings using the CLIP text encoder.

You only need to run it if:

* new labels are added
* label wording changes
* the vocabulary expands

Once embeddings are generated, Python is no longer required to run the application.

# How to Run the Project

Follow these steps.

---

# Step 1 Install Required Software

Install Visual Studio and select the workload:

.NET Desktop Development

---

Install Python and ensure the option

Add Python to PATH

is enabled.

Python is only required if you want to regenerate text embeddings.

---

# Step 2 Download the Repository

Clone the repository using Git:

git clone https://github.com/NegativeSolution/LearningWebcamThesisProject.git

Or download the ZIP file from GitHub and extract it.

---

# Step 3 Open the Project

1. Launch Visual Studio
2. Click Open a project or solution
3. Select

LearningWebcamThesisProject.slnx

---

# Step 4 Restore Dependencies

Visual Studio will automatically download required libraries from NuGet.

These include:

OpenCvSharp
Microsoft.ML.OnnxRuntime
System.Numerics.Tensors

If necessary you can manually run:

Build -> Restore NuGet Packages

---

# Step 5 Build the Program

Compile the program by selecting:

Build -> Build Solution

or pressing:

Ctrl + Shift + B

---

# Step 6 Run the Application

Press:

F5

or select:

Start Debugging

Your webcam window should appear.

---

# How to Use the Program

Once the webcam window opens:

Step 1
Move your mouse over an object.

Step 2
Click and drag to draw a bounding box around the object.

Step 3
Press the Spacebar.

The AI will:

1. Analyze the object
2. Compare it with all label embeddings
3. Select the most similar label
4. Save the image

Example console output:

Confidence: 0.87
Saved: a photo of a white ceramic mug

---

# Expanding the Object Vocabulary

You can add new recognizable objects.

Step 1
Edit:

Data/labels.txt

Add descriptions such as:

a photo of a bicycle
a photo of a keyboard
a photo of a backpack

---

Step 2
Regenerate embeddings:

python Scripts/generate_embeddings.py

---

Step 3
Rebuild the project.

The system will now recognize the new objects.

# Troubleshooting

Webcam does not open

Check that no other application is using the camera and that your device has a working webcam.

---

Model file not found

Verify that the file exists:

Models/clip_image_encoder.onnx

---

Embedding size mismatch

This occurs if embeddings were generated with a different model.

Regenerate embeddings using the provided script.

# Educational Purpose

This project demonstrates how modern AI systems connect vision and language representations.

Humans associate words with visual experiences.

CLIP uses a similar concept by learning a shared space where images and language can be compared numerically.

This allows the system to recognize objects it has never explicitly been trained on.
